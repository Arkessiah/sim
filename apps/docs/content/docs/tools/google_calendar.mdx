[Google Calendar](https://calendar.google.com) is Google's powerful calendar and scheduling service that provides a comprehensive platform for managing events, meetings, and appointments. With seamless integration across Google's ecosystem and widespread adoption, Google Calendar offers robust features for both personal and professional scheduling needs.

With Google Calendar, you can:

- **Create and manage events**: Schedule meetings, appointments, and reminders with detailed information
- **Send calendar invites**: Automatically notify and coordinate with attendees through email invitations
- **Natural language event creation**: Quickly add events using conversational language like "Meeting with John tomorrow at 3pm"
- **View and search events**: Easily find and access your scheduled events across multiple calendars
- **Manage multiple calendars**: Organize different types of events across various calendars

In Sim, the Google Calendar integration enables your agents to programmatically create, read, and manage calendar events. This allows for powerful automation scenarios such as scheduling meetings, sending calendar invites, checking availability, and managing event details. Your agents can create events with natural language input, send automated calendar invitations to attendees, retrieve event information, and list upcoming events. This integration bridges the gap between your AI workflows and calendar management, enabling seamless scheduling automation and coordination with one of the world's most widely used calendar platforms.

## Operations

### `google_calendar_create`

Create a new event in Google Calendar

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `calendarId` | string | No | Calendar ID \(defaults to primary\) |
| `summary` | string | Yes | Event title/summary |
| `description` | string | No | Event description |
| `location` | string | No | Event location |
| `startDateTime` | string | Yes | Start date and time \(RFC3339 format, e.g., 2025-06-03T10:00:00-08:00\) |
| `endDateTime` | string | Yes | End date and time \(RFC3339 format, e.g., 2025-06-03T11:00:00-08:00\) |
| `timeZone` | string | No | Time zone \(e.g., America/Los_Angeles\) |
| `attendees` | array | No | Array of attendee email addresses |
| `sendUpdates` | string | No | How to send updates to attendees: all, externalOnly, or none |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `content` | string | Event creation confirmation message |
| `metadata` | json | Created event metadata including ID, status, and details |

### `google_calendar_list`

List events from Google Calendar

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `calendarId` | string | No | Calendar ID \(defaults to primary\) |
| `timeMin` | string | No | Lower bound for events \(RFC3339 timestamp, e.g., 2025-06-03T00:00:00Z\) |
| `timeMax` | string | No | Upper bound for events \(RFC3339 timestamp, e.g., 2025-06-04T00:00:00Z\) |
| `orderBy` | string | No | Order of events returned \(startTime or updated\) |
| `showDeleted` | boolean | No | Include deleted events |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `content` | string | Summary of found events count |
| `metadata` | json | List of events with pagination tokens and event details |

### `google_calendar_get`

Get a specific event from Google Calendar

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `calendarId` | string | No | Calendar ID \(defaults to primary\) |
| `eventId` | string | Yes | Event ID to retrieve |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `content` | string | Event retrieval confirmation message |
| `metadata` | json | Event details including ID, status, times, and attendees |

### `google_calendar_quick_add`

Create events from natural language text

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `calendarId` | string | No | Calendar ID \(defaults to primary\) |
| `text` | string | Yes | Natural language text describing the event \(e.g., "Meeting with John tomorrow at 3pm"\) |
| `attendees` | array | No | Array of attendee email addresses \(comma-separated string also accepted\) |
| `sendUpdates` | string | No | How to send updates to attendees: all, externalOnly, or none |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `content` | string | Event creation confirmation message from natural language |
| `metadata` | json | Created event metadata including parsed details |

### `google_calendar_invite`

Invite attendees to an existing Google Calendar event

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `calendarId` | string | No | Calendar ID \(defaults to primary\) |
| `eventId` | string | Yes | Event ID to invite attendees to |
| `attendees` | array | Yes | Array of attendee email addresses to invite |
| `sendUpdates` | string | No | How to send updates to attendees: all, externalOnly, or none |
| `replaceExisting` | boolean | No | Whether to replace existing attendees or add to them \(defaults to false\) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `content` | string | Attendee invitation confirmation message with email delivery status |
| `metadata` | json | Updated event metadata including attendee list and details |



## Best Practices

{/* MANUAL-CONTENT-START:bestPractices */}
When integrating the Google Calendar block into your Sim.ai workflows, following established best practices ensures reliable event management, optimal performance, and seamless calendar automation. The Google Calendar block serves as a comprehensive scheduling hub that connects your automated workflows with Google's calendar infrastructure, enabling intelligent event creation, attendee management, and schedule monitoring.

**Connection Tag Usage and Data Flow**

Effective connection tag implementation is crucial for successful Google Calendar integrations. The `<google_calendar1.metadata>` output contains rich event data including event IDs, attendee lists, and timestamps that can be chained to subsequent blocks. Use `<google_calendar1.content>` for human-readable confirmations in notification systems, while leveraging the metadata for programmatic operations.

For dynamic event creation, connect AI agent outputs to calendar inputs using patterns like `<agent1.content>` â†’ `<google_calendar1.text>` for the quick_add operation, enabling natural language event scheduling. The `<google_calendar1.metadata.id>` field is particularly valuable for follow-up operations, allowing you to reference created events in subsequent invite or update operations.

When working with time-sensitive workflows, establish proper data validation by connecting date/time inputs from webhook payloads or form submissions directly to `<google_calendar1.startDateTime>` and `<google_calendar1.endDateTime>` fields. Ensure RFC3339 format compliance by using Function blocks to transform timestamps when necessary.

**Workflow Architecture Patterns**

Successful Google Calendar workflows follow several proven architectural patterns. The **Webhook â†’ Function â†’ Google Calendar** pattern excels for external system integrations, where incoming API calls trigger event creation with properly formatted datetime values and attendee lists.

For intelligent scheduling systems, implement **Agent â†’ Google Calendar Quick Add** patterns that leverage natural language processing. This approach allows users to input conversational text like "Schedule team standup every Monday at 9am" through chat interfaces or forms, with the quick_add operation parsing the intent automatically.

The **Schedule Trigger â†’ Google Calendar List â†’ Conditional Logic** pattern enables proactive calendar management. Use this for daily digest workflows that check upcoming events and send preparation reminders, or availability checking systems that list events within specific time ranges using `<google_calendar1.timeMin>` and `<google_calendar1.timeMax>` parameters.

For event management workflows, employ **Google Calendar Create â†’ Google Calendar Invite** sequences that first establish the event, then use the returned `<google_calendar1.metadata.id>` to add attendees dynamically. This pattern provides greater control over invitation timing and recipient management.

**Message/Data Formatting Excellence**

Proper datetime formatting is fundamental to Google Calendar block success. Always use RFC3339 format for `startDateTime` and `endDateTime` inputs: "2025-06-03T10:00:00-08:00". Include timezone information explicitly using the `timeZone` parameter to prevent scheduling conflicts across different geographic regions.

Structure attendee arrays consistently as JSON arrays of email strings: `["user1@company.com", "user2@company.com"]`. The block also accepts comma-separated strings for attendees, but array format provides better validation and error handling. When connecting from webhook data, use Function blocks to transform attendee data into the proper format.

For event descriptions, leverage HTML formatting to create rich, structured content. Include meeting agendas, dial-in information, and relevant links using proper HTML tags. This enhances the user experience and ensures calendar events contain actionable information.

Implement validation patterns for required fields before calendar operations. Use conditional logic to verify that `summary` and datetime fields contain valid data, preventing workflow failures due to malformed inputs.

**Debugging and Monitoring**

Establish comprehensive monitoring by tracking both `<google_calendar1.content>` and `<google_calendar1.metadata>` outputs. The content field provides immediate success/failure feedback, while metadata contains detailed operational results including event IDs, creation timestamps, and error details.

Monitor OAuth authentication status proactively. Google Calendar operations require valid OAuth tokens, and expired authentication will cause workflow failures. Implement error handling that detects authentication issues and provides clear user guidance for re-authorization.

Use the `showDeleted` parameter strategically in list operations for debugging purposes. When troubleshooting missing events, enable this option to reveal deleted events that might explain discrepancies in expected calendar data.

Track invitation delivery using the `sendUpdates` parameter effectively. Set to "all" for maximum visibility, "externalOnly" for mixed internal/external meetings, or "none" for draft events. Monitor the invitation status in metadata to ensure attendees receive proper notifications.

**Security Considerations**

Implement proper OAuth scope management by requesting only necessary calendar permissions. The Google Calendar block operates within the authenticated user's permission context, so ensure service accounts or OAuth applications have appropriate access levels for the target calendars.

Validate attendee email addresses before invitation to prevent information disclosure. Use Function blocks to sanitize and verify email formats, especially when processing attendee lists from external sources or user inputs.

Be mindful of calendar visibility when using non-primary calendar IDs. Events created in shared calendars inherit the calendar's sharing permissions, potentially exposing sensitive information to unintended audiences. Always verify `calendarId` parameters match intended privacy levels.

Consider rate limiting for high-volume calendar operations. Google Calendar API has usage quotas, and exceeding limits will cause workflow failures. Implement appropriate delays or batching strategies for workflows processing multiple events simultaneously.

**Performance Optimization**

Optimize datetime operations by standardizing timezone handling across workflows. Use consistent timezone parameters or UTC timestamps to minimize conversion overhead and prevent scheduling errors. Store commonly used timezone values as workflow constants for reusability.

Leverage the quick_add operation for user-facing scheduling interfaces due to its natural language processing capabilities, but use standard create operations for programmatic scheduling where exact control over all parameters is required. The quick_add parsing introduces slight latency but significantly improves user experience.

Implement intelligent caching for calendar list operations by using appropriate `timeMin` and `timeMax` boundaries. Avoid open-ended queries that retrieve excessive historical data, which increases response times and API quota consumption.

Batch attendee operations efficiently by using the invite operation strategically. Rather than creating events with all attendees immediately, create the base event first, then use targeted invite operations for different attendee groups, enabling better control over invitation timing and reducing the risk of delivery failures affecting event creation.

Use the `orderBy` parameter in list operations to optimize result processing. Order by "startTime" for chronological workflows and "updated" for change-detection systems, ensuring your workflow receives data in the most useful sequence for subsequent processing steps.

  <Accordion title="Why is my Google Calendar event creation failing with authentication errors?">

    Google Calendar requires OAuth 2.0 authentication setup in Sim.ai:

    #### Authentication Setup Steps

    1. **Enable Google Calendar API** in your Google Cloud Console
    2. **Create OAuth 2.0 credentials** with appropriate scopes
    3. **Configure the integration** in Sim.ai settings
    4. **Grant calendar permissions** during the OAuth flow

    #### Required OAuth Scopes
    - `https://www.googleapis.com/auth/calendar` - Full calendar access
    - `https://www.googleapis.com/auth/calendar.events` - Event management only

    #### Common Authentication Issues

    | Error | Cause | Solution |
    |-------|-------|----------|
    | `Invalid credentials` | OAuth token expired | Re-authenticate in Sim.ai settings |
    | `Insufficient permissions` | Missing calendar scopes | Update OAuth scope configuration |
    | `Calendar not found` | Invalid `calendarId` | Use `primary` or verify calendar ID |
