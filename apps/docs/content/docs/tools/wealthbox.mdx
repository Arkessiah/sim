[Wealthbox](https://www.wealthbox.com/) is a comprehensive CRM platform designed specifically for financial advisors and wealth management professionals. It provides a centralized system for managing client relationships, tracking interactions, and organizing business workflows in the financial services industry.

With Wealthbox, you can:

- **Manage client relationships**: Store detailed contact information, background data, and relationship histories for all your clients
- **Track interactions**: Create and maintain notes about meetings, calls, and other client touchpoints
- **Organize tasks**: Schedule and manage follow-up activities, deadlines, and important action items
- **Document workflows**: Keep comprehensive records of client communications and business processes
- **Access client data**: Retrieve information quickly with organized contact management and search capabilities
- **Automate follow-ups**: Set reminders and schedule tasks to ensure consistent client engagement

In Sim, the Wealthbox integration enables your agents to seamlessly interact with your CRM data through OAuth authentication. This allows for powerful automation scenarios such as automatically creating client notes from meeting transcripts, updating contact information, scheduling follow-up tasks, and retrieving client details for personalized communications. Your agents can read existing notes, contacts, and tasks to understand client history, while also creating new entries to maintain up-to-date records. This integration bridges the gap between your AI workflows and your client relationship management, enabling automated data entry, intelligent client insights, and streamlined administrative processes that free up time for more valuable client-facing activities.

## Operations

### `wealthbox_read_note`

Read content from a Wealthbox note

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `noteId` | string | No | The ID of the note to read |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `success` | boolean | Operation success status |
| `output` | object | Note data and metadata |

### `wealthbox_write_note`

Create or update a Wealthbox note

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `content` | string | Yes | The main body of the note |
| `contactId` | string | No | ID of contact to link to this note |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `success` | boolean | Operation success status |
| `output` | object | Created or updated note data and metadata |

### `wealthbox_read_contact`

Read content from a Wealthbox contact

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `contactId` | string | No | The ID of the contact to read |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `success` | boolean | Operation success status |
| `output` | object | Contact data and metadata |

### `wealthbox_write_contact`

Create a new Wealthbox contact

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `firstName` | string | Yes | The first name of the contact |
| `lastName` | string | Yes | The last name of the contact |
| `emailAddress` | string | No | The email address of the contact |
| `backgroundInformation` | string | No | Background information about the contact |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `success` | boolean | Operation success status |
| `output` | object | Created or updated contact data and metadata |

### `wealthbox_read_task`

Read content from a Wealthbox task

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `taskId` | string | No | The ID of the task to read |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `success` | boolean | Operation success status |
| `output` | object | Task data and metadata |

### `wealthbox_write_task`

Create or update a Wealthbox task

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `title` | string | Yes | The name/title of the task |
| `dueDate` | string | Yes | The due date and time of the task \(format: "YYYY-MM-DD HH:MM AM/PM -HHMM", e.g., "2015-05-24 11:00 AM -0400"\) |
| `contactId` | string | No | ID of contact to link to this task |
| `description` | string | No | Description or notes about the task |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `success` | boolean | Operation success status |
| `output` | object | Created or updated task data and metadata |



## Best Practices

{/* MANUAL-CONTENT-START:bestPractices */}
When integrating the Wealthbox block into your Sim.ai workflows, following established best practices ensures reliable CRM operations and optimal data management. The Wealthbox block serves as a comprehensive interface for financial advisory firms to automate client relationship management tasks, enabling seamless synchronization between your workflows and Wealthbox's contact, note, and task management systems.

**Connection Tag Usage and Data Flow**

Proper connection tag implementation is critical for effective Wealthbox integrations. The `<wealthbox1.contact>` connection tag provides rich client data that can be forwarded to subsequent blocks for analysis or reporting. When creating notes, use `<agent1.content>` to populate the `content` parameter, enabling AI-generated meeting summaries or client interaction records to be automatically stored in Wealthbox.

For task management workflows, leverage the `<wealthbox1.task>` output to chain task creation with follow-up actions. The `<wealthbox1.metadata>` connection tag contains essential operation details including newly created item IDs, which are crucial for subsequent read operations. Always validate the `<wealthbox1.success>` output before proceeding with dependent operations to ensure data integrity throughout your workflow.

When linking related entities, use the `contactId` parameter strategically. Extract contact IDs from `<wealthbox1.contact>` outputs using JSONPath expressions to properly associate notes and tasks with specific clients, maintaining relationship integrity in your CRM system.

**Workflow Architecture Patterns**

Successful Wealthbox workflows typically follow established patterns that align with financial advisory business processes. The **Webhook → Agent → Wealthbox** pattern excels for meeting transcription workflows, where incoming audio data triggers AI processing to generate structured notes that are automatically saved to client records.

For client onboarding, implement **Form Trigger → Wealthbox Write Contact → Wealthbox Write Task** patterns, where new client information creates contact records and generates initial follow-up tasks. This ensures no new client falls through the cracks and maintains consistent onboarding processes.

Task management workflows benefit from **Calendar → Agent → Wealthbox Write Task** patterns, where calendar events trigger AI analysis to create relevant follow-up tasks with appropriate due dates. For compliance workflows, consider **Email → Function → Wealthbox Write Note** patterns that automatically document client communications and regulatory interactions.

Advanced implementations use **Wealthbox Read → Agent → Multiple Wealthbox Writes** patterns for client portfolio reviews, where existing contact data triggers AI analysis that generates comprehensive notes and follow-up tasks across multiple client accounts.

**Message/Data Formatting Excellence**

Date formatting requires precise attention in Wealthbox integrations. The `dueDate` parameter demands the exact format "YYYY-MM-DD HH:MM AM/PM -HHMM" (e.g., "2024-12-15 02:30 PM -0500"). Use Function blocks to transform date inputs from other systems into this specific format before passing to Wealthbox write operations.

Structure note content for maximum readability in the Wealthbox interface. Use clear headings, bullet points, and paragraph breaks in the `content` parameter. For meeting notes, implement consistent templates: client concerns, action items, next steps, and follow-up dates. This standardization improves team collaboration and client service quality.

When creating contacts, populate optional fields like `backgroundInformation` and `emailAddress` whenever available. Rich contact profiles enable more effective client relationship management and improve the quality of AI-generated insights in subsequent workflow operations.

For task descriptions, include specific action items and context. Use the `description` parameter to provide enough detail for team members to complete tasks without additional clarification, improving operational efficiency.

**Debugging and Monitoring**

Implement comprehensive error handling by monitoring the `<wealthbox1.success>` output in all workflows. Create conditional branches that handle authentication failures, network timeouts, and data validation errors gracefully. The Wealthbox API requires valid OAuth tokens, so monitor for authentication-related failures and implement token refresh mechanisms.

Use the `<wealthbox1.metadata>` output for detailed operation tracking. This includes item IDs for newly created records, operation timestamps, and error details when operations fail. Implement logging workflows that capture this metadata for audit trails and compliance reporting.

For read operations, validate that returned data structures match your expectations before processing. Empty results from read operations don't necessarily indicate failures - they may represent valid empty states that require different handling logic.

Monitor rate limiting and API quotas by tracking operation frequency and response times. Implement exponential backoff strategies for retries and consider batch processing for high-volume operations to optimize API usage.

**Security Considerations**

Wealthbox blocks handle sensitive financial and personal client data, requiring strict security protocols. Ensure OAuth tokens are stored securely and never logged in plain text. Implement token rotation policies and monitor for unauthorized access attempts through the `<wealthbox1.success>` status monitoring.

Validate all input data before sending to Wealthbox operations. Sanitize text content in notes and task descriptions to prevent potential security issues. Use data validation blocks to ensure email addresses, dates, and other structured data meet expected formats before processing.

Implement access control at the workflow level to ensure only authorized users can trigger Wealthbox operations. For multi-tenant environments, include contact ID validation to prevent cross-client data access violations.

Consider data residency requirements for financial services compliance. Monitor which data flows through your workflows and ensure sensitive client information handling meets regulatory requirements for your jurisdiction.

**Performance Optimization**

Batch related operations whenever possible to reduce API calls and improve response times. Instead of creating individual notes for each client interaction, aggregate related content and create comprehensive notes that cover multiple interaction points.

Cache frequently accessed contact and task data using workflow variables to minimize redundant read operations. This is particularly effective for workflows that reference the same client records multiple times within a single execution.

For large-scale operations, implement pagination handling for bulk data retrieval. The Wealthbox API may return paginated results for extensive contact or task lists, requiring iterative processing to capture complete datasets.

Optimize date and time handling by standardizing timezone conversions early in your workflows. Convert all temporal data to the required Wealthbox format once, then reuse the formatted values across multiple operations to reduce processing overhead.

Use conditional logic to skip unnecessary operations. For example, check if a contact already exists before attempting creation, or verify task completion status before creating duplicate follow-up tasks. This reduces API load and prevents data duplication issues.
