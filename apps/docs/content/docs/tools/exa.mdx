[Exa](https://exa.ai/) is an AI-powered search engine designed specifically for developers and researchers, providing highly relevant and up-to-date information from across the web. It combines advanced semantic search capabilities with AI understanding to deliver more accurate and contextually relevant results than traditional search engines.

With Exa, you can:

- **Search with natural language**: Find information using conversational queries and questions
- **Get precise results**: Receive highly relevant search results with semantic understanding
- **Access up-to-date information**: Retrieve current information from across the web
- **Find similar content**: Discover related resources based on content similarity
- **Extract webpage contents**: Retrieve and process the full text of web pages
- **Answer questions with citations**: Ask questions and receive direct answers with supporting sources
- **Perform research tasks**: Automate multi-step research workflows to gather, synthesize, and summarize information

In Sim, the Exa integration allows your agents to search the web for information, retrieve content from specific URLs, find similar resources, answer questions with citations, and conduct research tasksâ€”all programmatically through API calls. This enables your agents to access real-time information from the internet, enhancing their ability to provide accurate, current, and relevant responses. The integration is particularly valuable for research tasks, information gathering, content discovery, and answering questions that require up-to-date information from across the web.

## Operations

### `exa_search`

Search the web using Exa AI. Returns relevant search results with titles, URLs, and text snippets.

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `query` | string | Yes | The search query to execute |
| `numResults` | number | No | Number of results to return \(default: 10, max: 25\) |
| `useAutoprompt` | boolean | No | Whether to use autoprompt to improve the query \(default: false\) |
| `type` | string | No | Search type: neural, keyword, auto or fast \(default: auto\) |
| `apiKey` | string | Yes | Exa AI API Key |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `results` | array | Search results with titles, URLs, and text snippets |

### `exa_get_contents`

Retrieve the contents of webpages using Exa AI. Returns the title, text content, and optional summaries for each URL.

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `urls` | string | Yes | Comma-separated list of URLs to retrieve content from |
| `text` | boolean | No | If true, returns full page text with default settings. If false, disables text return. |
| `summaryQuery` | string | No | Query to guide the summary generation |
| `apiKey` | string | Yes | Exa AI API Key |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `results` | array | Retrieved content from URLs with title, text, and summaries |

### `exa_find_similar_links`

Find webpages similar to a given URL using Exa AI. Returns a list of similar links with titles and text snippets.

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `url` | string | Yes | The URL to find similar links for |
| `numResults` | number | No | Number of similar links to return \(default: 10, max: 25\) |
| `text` | boolean | No | Whether to include the full text of the similar pages |
| `apiKey` | string | Yes | Exa AI API Key |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `similarLinks` | array | Similar links found with titles, URLs, and text snippets |

### `exa_answer`

Get an AI-generated answer to a question with citations from the web using Exa AI.

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `query` | string | Yes | The question to answer |
| `text` | boolean | No | Whether to include the full text of the answer |
| `apiKey` | string | Yes | Exa AI API Key |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `answer` | string | AI-generated answer to the question |
| `citations` | array | Sources and citations for the answer |

### `exa_research`

Perform comprehensive research using AI to generate detailed reports with citations

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `query` | string | Yes | Research query or topic |
| `includeText` | boolean | No | Include full text content in results |
| `apiKey` | string | Yes | Exa AI API Key |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `research` | array | Comprehensive research findings with citations and summaries |



## Best Practices

{/* MANUAL-CONTENT-START:bestPractices */}
When implementing the Exa block in your Sim.ai workflows, following proven best practices ensures optimal search performance and reliable data integration. The Exa block serves as your gateway to AI-powered web search, content extraction, and research capabilities, transforming raw queries into structured, actionable insights that seamlessly integrate with downstream workflow components.

**Connection Tag Usage and Data Flow**

Effective connection tag usage with the Exa block requires understanding its distinct output structures for each operation type. The `<exa1.results>` connection tag contains structured search data perfect for feeding into Agent blocks via `<agent1.context>`, enabling AI analysis of search findings. For research workflows, connect `<exa1.research>` to Function blocks using `<function1.input>` to process comprehensive findings programmatically.

When chaining Exa operations, leverage the `<exa1.results[0].url>` pattern to extract specific URLs from search results and pipe them into subsequent `exa_get_contents` operations. The `<exa1.answer>` output pairs exceptionally well with `<agent1.systemPrompt>` connections, allowing AI agents to build upon Exa's research findings with additional context and analysis.

For similarity searches, connect `<exa1.similarLinks>` to conditional logic blocks, enabling workflows that automatically filter and categorize related content based on relevance scores or domain patterns.

**Workflow Architecture Patterns**

Successful Exa implementations follow several proven architectural patterns. The **Research Pipeline** pattern combines multiple Exa operations: **Exa Search â†’ Exa Get Contents â†’ Agent Analysis**, where initial searches identify relevant URLs, content extraction retrieves full text, and AI agents synthesize findings into actionable insights.

The **Competitive Intelligence** pattern leverages **Exa Find Similar Links â†’ Content Extraction â†’ Data Aggregation** flows, automatically discovering competitor content and extracting strategic insights. This pattern excels for market research and trend analysis workflows.

For real-time information workflows, implement **Webhook Trigger â†’ Exa Answer â†’ Response Formatting** patterns, where external queries trigger immediate AI-powered answers with citations. This creates powerful Q&A systems backed by current web information.

The **Content Discovery** pattern utilizes **Exa Search â†’ URL Filtering â†’ Batch Content Retrieval**, enabling systematic content collection across multiple domains while maintaining quality control through intermediate filtering steps.

**Message/Data Formatting Excellence**

Optimize Exa query formatting to maximize search relevance and accuracy. Structure research queries using specific terminology and domain expertise rather than generic keywords. For the `useAutoprompt` parameter, enable it for complex, multi-faceted queries where Exa's query enhancement improves result quality, but disable it for precise, technical searches where exact terminology matters.

Configure the `numResults` parameter based on downstream processing capacity. Use lower values (5-10) for Agent processing to prevent context window overflow, and higher values (15-25) for Function blocks that can handle larger datasets efficiently. The `type` parameter should align with your search intent: use "neural" for conceptual searches, "keyword" for exact matches, and "auto" for balanced results.

When formatting `summaryQuery` inputs for content retrieval, provide specific guidance about desired summary focus areas. Instead of generic requests, use targeted queries like "key technical specifications" or "pricing and availability information" to generate more useful summaries.

Structure URL inputs for batch operations using consistent formatting with proper comma separation and validation. Implement error handling for malformed URLs to prevent entire operation failures due to single invalid inputs.

**Debugging and Monitoring**

Implement comprehensive monitoring for Exa operations by tracking result quality metrics and response patterns. Monitor the `<exa1.results>` array length to ensure searches return expected result counts, and implement alerts when result counts drop below baseline thresholds, indicating potential query or API issues.

Track citation quality in `<exa1.citations>` outputs by monitoring source diversity and relevance scores. Low-quality citation patterns often indicate suboptimal query formulation or API configuration issues that require investigation.

For content retrieval operations, monitor text length variations in `<exa1.results>` to identify pages that fail to load properly or return minimal content. Implement retry logic for URLs that consistently return low-quality content or encounter extraction errors.

Use structured logging to track query patterns and result quality over time. This enables identification of degrading search performance and optimization opportunities for query refinement and parameter tuning.

**Security Considerations**

Implement secure API key management by storing Exa API keys in encrypted environment variables rather than hardcoding them in workflow configurations. Use Sim.ai's secure credential management features to protect API keys across development and production environments.

Validate and sanitize all user-provided queries before passing them to Exa operations. Implement input filtering to prevent potential injection attacks or malformed queries that could compromise workflow stability or expose sensitive information.

For workflows processing sensitive research topics, implement result filtering to exclude potentially problematic content sources. Monitor citation domains and implement blocklists for unreliable or inappropriate content sources that don't meet your organization's standards.

Consider rate limiting and usage monitoring to prevent unexpected API quota consumption, especially in workflows triggered by external events or user interactions.

**Performance Optimization**

Optimize Exa block performance through strategic operation selection and parameter tuning. Use `exa_search` with focused queries for initial discovery, then leverage `exa_get_contents` only for the most relevant URLs rather than processing all search results. This reduces API calls and improves overall workflow performance.

Implement intelligent caching strategies for frequently requested content by storing `<exa1.results>` outputs in workflow variables or external storage systems. This prevents redundant API calls for duplicate queries within reasonable time windows.

Configure parallel processing for batch operations when retrieving content from multiple URLs. Structure workflows to process URL batches concurrently while respecting API rate limits and avoiding overwhelming downstream processing components.

Optimize the `text` parameter usage based on downstream requirements. Disable text retrieval when only metadata is needed, and enable it selectively for content that requires full analysis. This reduces bandwidth usage and improves response times.

For research-intensive workflows, implement progressive refinement patterns where initial broad searches inform more targeted subsequent queries, reducing overall API consumption while improving result relevance and quality.
