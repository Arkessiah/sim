[ArXiv](https://arxiv.org/) is a free, open-access repository of scientific research papers in fields such as physics, mathematics, computer science, quantitative biology, quantitative finance, statistics, electrical engineering, systems science, and economics. ArXiv provides a vast collection of preprints and published articles, making it a primary resource for researchers and practitioners worldwide.

With ArXiv, you can:

- **Search for academic papers**: Find research by keywords, author names, titles, categories, and more
- **Retrieve paper metadata**: Access abstracts, author lists, publication dates, and other bibliographic information
- **Download full-text PDFs**: Obtain the complete text of most papers for in-depth study
- **Explore author contributions**: View all papers by a specific author
- **Stay up-to-date**: Discover the latest submissions and trending topics in your field

In Sim, the ArXiv integration enables your agents to programmatically search, retrieve, and analyze scientific papers from ArXiv. This allows you to automate literature reviews, build research assistants, or incorporate up-to-date scientific knowledge into your agentic workflows. Use ArXiv as a dynamic data source for research, discovery, and knowledge extraction within your Sim projects.

## Operations

### `arxiv_search`

Search for academic papers on ArXiv by keywords, authors, titles, or other fields.

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `searchQuery` | string | Yes | The search query to execute |
| `searchField` | string | No | Field to search in: all, ti \(title\), au \(author\), abs \(abstract\), co \(comment\), jr \(journal\), cat \(category\), rn \(report number\) |
| `maxResults` | number | No | Maximum number of results to return \(default: 10, max: 2000\) |
| `sortBy` | string | No | Sort by: relevance, lastUpdatedDate, submittedDate \(default: relevance\) |
| `sortOrder` | string | No | Sort order: ascending, descending \(default: descending\) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `papers` | json | Array of papers matching the search query |

### `arxiv_get_paper`

Get detailed information about a specific ArXiv paper by its ID.

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `paperId` | string | Yes | ArXiv paper ID \(e.g., "1706.03762"\) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `paper` | json | Detailed information about the requested ArXiv paper |

### `arxiv_get_author_papers`

Search for papers by a specific author on ArXiv.

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `authorName` | string | Yes | Author name to search for |
| `maxResults` | number | No | Maximum number of results to return \(default: 10, max: 2000\) |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `authorPapers` | json | Array of papers authored by the specified author |



## Best Practices

{/* MANUAL-CONTENT-START:bestPractices */}
When integrating the ArXiv block into your Sim.ai workflows, following established best practices ensures reliable academic research retrieval and optimal performance. The ArXiv block serves as a gateway to accessing the vast collection of scientific papers on ArXiv, enabling automated research workflows, literature reviews, and knowledge discovery systems.

**Connection Tag Usage and Data Flow**

Proper use of connection tags is fundamental to successful ArXiv integrations. The `<arxiv1.papers>` connection tag represents the primary output for search operations and should be connected to downstream blocks that process research data. When using `<arxiv1.paper>` for specific paper retrieval, ensure the connection flows to blocks expecting detailed metadata structures including abstracts, authors, and publication dates.

The `<arxiv1.totalResults>` output provides valuable workflow control information. Connect this to conditional logic blocks to handle cases where searches return no results or exceed expected thresholds. For author-specific searches, `<arxiv1.authorPapers>` contains structured data that can be mapped directly to database storage blocks or analysis tools.

Critical data flow patterns include chaining search operations where `<searchBlock.papers[0].id>` feeds into `<arxiv2.paperId>` for detailed retrieval. This enables workflows that first discover relevant papers through broad searches, then fetch comprehensive metadata for selected results.

**Workflow Architecture Patterns**

Successful ArXiv workflows typically follow established architectural patterns. The most common pattern involves **Trigger → ArXiv Search → Agent Analysis → Output**, where external triggers initiate research queries, ArXiv returns results, and AI agents analyze the findings. This pattern enables automated literature monitoring systems that track new publications in specific research areas.

For comprehensive research workflows, implement **ArXiv Search → Filter/Sort → ArXiv Get Paper → Database Storage** patterns. This architecture first identifies relevant papers through broad searches, applies filtering logic to select the most pertinent results, retrieves detailed information, and stores structured data for future analysis.

Advanced patterns include **Scheduled Trigger → Author Search → Comparison Logic → Notification** flows for monitoring specific researchers' publications. These workflows automatically track new papers from followed authors and alert users to relevant publications in their field of interest.

**Message/Data Formatting Excellence**

ArXiv search queries require careful formatting for optimal results. Structure your `<arxiv1.searchQuery>` inputs using appropriate boolean operators and field-specific syntax. For title searches, use precise terminology and consider alternative phrasings. When searching abstracts, include domain-specific keywords and technical terms relevant to your research area.

The `<arxiv1.searchField>` parameter enables targeted searches that significantly improve result relevance. Use "ti" for title-specific searches when looking for papers on exact topics, "au" for author searches, and "abs" for concept-based discovery through abstract content. Combine multiple search strategies in parallel workflows to ensure comprehensive coverage.

Format `<arxiv1.maxResults>` strategically based on downstream processing capabilities. For manual review workflows, limit results to 20-50 papers. For automated analysis systems, consider higher limits but implement proper error handling for large datasets. The 2000-result maximum should be used judiciously to avoid performance degradation.

**Debugging and Monitoring**

Enable comprehensive logging for all ArXiv blocks in your workflows. Monitor the `<arxiv1.totalResults>` output to track search effectiveness and identify queries that return insufficient or excessive results. Use this data to refine search strategies and optimize query parameters over time.

Implement error handling for cases where `<arxiv1.papers>` returns empty arrays or `<arxiv1.paper>` fails to retrieve specific documents. ArXiv paper IDs must follow exact formatting (e.g., "1706.03762" or "cs.AI/0001001"), and malformed IDs will cause operation failures. Validate paper ID formats before passing them to retrieval operations.

Monitor API response times and implement timeout handling for large result sets. ArXiv's servers may experience delays during peak usage periods, particularly for complex searches or requests for popular papers. Set appropriate timeout values and implement retry logic for critical research workflows.

**Performance Optimization**

Optimize ArXiv block performance by implementing intelligent caching strategies. Store frequently accessed paper metadata locally using database blocks to reduce redundant API calls. Cache search results for common queries and implement time-based expiration to balance performance with data freshness.

Structure search queries efficiently by using the most restrictive search fields first. Title and author searches typically return faster results than abstract or full-text searches. When possible, combine multiple specific criteria rather than relying on broad keyword searches that require extensive server-side processing.

Implement pagination strategies for large research datasets. Rather than requesting maximum results in single operations, design workflows that retrieve papers in batches and process them incrementally. This approach reduces memory usage and improves overall workflow reliability.

**Security Considerations**

Validate all user-provided search inputs to prevent injection attacks or malformed queries that could cause system instability. Implement input sanitization for search terms, especially when accepting queries from external sources or user interfaces.

Consider rate limiting when implementing public-facing research tools that utilize the ArXiv block. Excessive API usage can impact service availability and may violate ArXiv's usage policies. Implement request throttling and user session management to ensure responsible API consumption.

Store retrieved research data securely, particularly when building proprietary research databases or commercial applications. While ArXiv content is generally open access, implement appropriate data handling practices and respect any usage restrictions specified by individual papers or authors.

### How do I search for academic papers using the ArXiv block in my workflow?

To search for academic papers from ArXiv, add an **ArXiv block** to your Sim.ai workflow and configure the `arxiv_search` operation with these inputs:

#### Required Inputs

| Input | Description | Example |
|-------|-------------|---------|
| `searchQuery` | Keywords, topics, or research terms | `<start.topic>` or static text like "machine learning" |

#### Optional Search Parameters

| Input | Description | Options | Example |
|-------|-------------|---------|---------|
| `searchField` | Specific field to search | all, ti (title), au (author), abs (abstract), co (comment), jr (journal), cat (category), rn (report number) | `<start.searchField>` |
| `maxResults` | Number of papers to return | 1-2000 (default: 10) | `<start.maxResults>` |
| `sortBy` | Sort criteria | relevance, lastUpdatedDate, submittedDate | `<start.sortBy>` |
| `sortOrder` | Sort direction | ascending, descending | `<start.sortOrder>` |

### What common errors occur when using the ArXiv block and how do I fix them?

Troubleshoot ArXiv integration issues with these solutions:

#### Common Issues and Resolutions

| Error Type | Cause | Solution |
|------------|-------|----------|
| **Invalid Paper ID Format** | Wrong ID format provided | Use format like `"1706.03762"` without "arXiv:" prefix |
| **Empty Search Results** | Query too specific or typos | Broaden search terms in `<arxiv1.searchQuery>` |
| **Rate Limiting** | Too many requests | Reduce `maxResults` or add delays between requests |
| **Author Name Not Found** | Exact name match required | Try variations: "Smith, J." vs "John Smith" |
| **Connection Tag Errors** | Incorrect output references | Verify tags like `<arxiv1.papers>` match block name |

Check workflow execution logs for specific API error messages from ArXiv's service.
