## Webhook

Trigger workflows from new Telegram messages that mention or DM your bot. You can use the Telegram block in trigger mode to start a workflow when your bot receives updates from Telegram via a webhook. Click the lightning bolt icon to enable trigger mode, then enter your bot token and click save changes.

### Configuration Inputs

| Input | Type | Required | Description |
|------|------|----------|-------------|
| `allowedUpdates` | array | No | List of update types to receive (e.g., `["message"]`). Defaults to all. |
| `secretToken` | string | No | Secret token for verifying Telegram webhook requests (`X-Telegram-Bot-Api-Secret-Token`). |
| `includeEntities` | boolean | No | Include parsed entities (mentions, commands) in outputs. Defaults to `true`. |

### Outputs

| Output | Type | Description |
|--------|------|-------------|
| `payloads` | array | The raw Telegram update payloads received. |
| `latestPayload` | object | Convenience object of the latest update. |
| `telegramMessage` | object | Parsed message fields (chat, from, text, entities). |

#### `latestPayload` shape

| Field | Type | Description |
|-------|------|-------------|
| `update_id` | number | Unique identifier for the update. |
| `message.message_id` | number | Unique message ID. |
| `message.date` | number | Unix time when the message was sent. |
| `message.chat` | object | Chat the message belongs to (id, type, title). |
| `message.from` | object | Sender user object. |
| `message.text` | string | Message text. |
| `message.entities` | array | Message entities (bot commands, mentions). |

### Example webhook event body (raw Telegram update payload)

```json
{
  "update_id": 10000,
  "message": {
    "message_id": 1365,
    "date": 1441645532,
    "chat": { "id": -1001234567890, "type": "supergroup", "title": "My Group" },
    "from": { "id": 1111111, "is_bot": false, "first_name": "Alice" },
    "text": "/start hello",
    "entities": [ { "offset": 0, "length": 6, "type": "bot_command" } ]
  }
}
```

Method: `POST`  •  Headers: `Content-Type: application/json`, `X-Telegram-Bot-Api-Secret-Token` (optional)

## Operations

### `telegram_message`

Send messages to Telegram channels or users through the Telegram Bot API. Enables direct communication and notifications with message tracking and chat confirmation.

#### Input

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `botToken` | string | Yes | Your Telegram Bot API Token |
| `chatId` | string | Yes | Target Telegram chat ID |
| `text` | string | Yes | Message text to send |

#### Output

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `success` | boolean | Telegram message send success status |
| `messageId` | number | Unique Telegram message identifier |
| `chatId` | string | Target chat ID where message was sent |
| `text` | string | Text content of the sent message |
| `timestamp` | number | Unix timestamp when message was sent |
| `from` | object | Information about the bot that sent the message |



## Best Practices

{/* MANUAL-CONTENT-START:bestPractices */}
When integrating the Telegram block into your Sim.ai workflows, following established best practices ensures reliable message delivery, optimal performance, and secure communication channels. The Telegram block serves as a versatile communication bridge between your automated workflows and Telegram's messaging platform, enabling real-time notifications, interactive bot responses, and comprehensive webhook handling for bidirectional communication.

**Connection Tag Usage and Data Flow**

Proper connection tag implementation is fundamental to successful Telegram integrations. The `<telegram1.success>` output provides critical feedback for conditional workflow branching, allowing you to implement retry logic when message delivery fails. Use `<telegram1.messageId>` to track individual messages across complex workflows, particularly valuable for building conversation threads or implementing message editing functionality.

For webhook-triggered workflows, leverage the comprehensive message context through tags like `<telegram1.from_username>`, `<telegram1.chat_type>`, and `<telegram1.text>` to create intelligent routing patterns. The `<telegram1.entities>` output contains structured data about mentions, hashtags, and commands, enabling sophisticated message parsing for bot interactions.

When chaining Telegram blocks with Agent blocks, use `<agent1.content>` to pipe AI-generated responses directly into the `text` input. For dynamic chat targeting, connect database outputs to the `chatId` input using tags like `<database1.telegram_chat_id>` to route messages to appropriate channels based on workflow conditions.

**Workflow Architecture Patterns**

The **Webhook → Telegram** pattern forms the foundation for notification systems. Structure these workflows to capture external events and translate them into formatted Telegram messages. For complex notifications, implement **API → Function → Telegram** chains where the Function block processes raw data into user-friendly message formats.

Interactive bot architectures follow **Telegram Webhook → Agent → Telegram** patterns, creating conversational interfaces that process natural language inputs and deliver AI-generated responses. Enhance these patterns with **Telegram → Database → Telegram** flows for persistent conversation memory and user preference tracking.

For multi-channel communication, implement **Trigger → Multiple Telegram** patterns where a single event broadcasts different message formats to various chat types. Use conditional blocks to customize message content based on `<telegram1.chatType>` outputs, ensuring appropriate formatting for private chats versus group channels.

**Message/Data Formatting Excellence**

Telegram's rich text formatting capabilities require careful attention to message structure. Implement proper Markdown formatting for enhanced readability: use `*bold*`, `_italic_`, and `code` syntax appropriately. Structure complex data using bullet points and numbered lists to improve message comprehension in mobile environments.

For automated notifications, establish consistent message templates that include contextual information. Incorporate timestamps using `<telegram1.timestamp>` outputs for audit trails, and structure error messages to include actionable troubleshooting steps. When sending data from APIs or databases, filter and format content to prevent information overload in chat environments.

Handle special characters and Unicode properly by validating text inputs before transmission. Implement character count limits proactively, as Telegram enforces message length restrictions that can cause delivery failures if exceeded.

**Debugging and Monitoring**

Implement comprehensive monitoring using the `<telegram1.ok>` output to track API response success rates. Create conditional branches that handle failed message deliveries by checking `<telegram1.success>` values and implementing appropriate retry mechanisms or alternative notification channels.

Monitor chat accessibility by tracking `<telegram1.chatId>` outputs and maintaining logs of successful delivery confirmations. Bot token authentication failures typically manifest as consistent false values in success outputs, indicating the need for token validation or renewal.

For webhook debugging, examine the complete `<telegram1.result>` object to understand message structure and identify parsing issues. Common debugging scenarios include mismatched chat IDs, expired bot tokens, and insufficient bot permissions for target channels.

Establish monitoring dashboards that track message delivery rates, response times, and error patterns. Use `<telegram1.messageDate>` timestamps to identify delivery delays and optimize workflow performance accordingly.

**Security Considerations**

Protect your Telegram bot tokens as sensitive credentials within Sim.ai's secure environment variables. Never hardcode bot tokens directly into workflow configurations, instead referencing them through encrypted storage mechanisms to prevent unauthorized access.

Implement chat ID validation to ensure messages reach intended recipients. For public bot deployments, establish authentication mechanisms that verify user permissions before processing sensitive commands or delivering confidential information through Telegram channels.

When handling webhook data, sanitize and validate all inputs from `<telegram1.text>` and `<telegram1.entities>` to prevent injection attacks or malformed data from disrupting workflow execution. Implement rate limiting for bot interactions to prevent abuse and maintain service availability.

Consider message content sensitivity when routing information through Telegram channels. For compliance-sensitive environments, implement content filtering and audit logging to ensure appropriate data handling across all Telegram communications.

**Performance Optimization**

Optimize workflow performance by batching related operations and minimizing redundant API calls. Cache frequently accessed chat IDs and user information to reduce lookup overhead in high-volume messaging scenarios.

Implement intelligent retry mechanisms that respect Telegram's rate limiting policies. Structure retry logic with exponential backoff patterns, using `<telegram1.success>` outputs to trigger appropriate delay intervals before retransmission attempts.

For bulk messaging operations, implement queue-based patterns that process messages sequentially rather than simultaneously, preventing API rate limit violations that could disrupt entire workflow execution. Monitor `<telegram1.timestamp>` outputs to track processing speeds and identify bottlenecks in complex workflows.

Optimize message content by removing unnecessary formatting and redundant information that increases transmission overhead. Structure workflows to aggregate related notifications into single messages rather than sending multiple individual alerts, improving both performance and user experience.

    - `<start.from_username>` - Username of the message sender

  <Accordion title="How do I connect an Agent block's response to send as a Telegram message?">

    To route an AI agent's response directly to Telegram:

    1. **In the Telegram block's `text` input**, type `<>` to open the connection menu
    2. **Select the agent's output** like `<agent1.content>` from available connections

    #### Example Chatbot Configuration
    ```
    Webhook → Agent → Telegram
    ```

    - Agent `userPrompt`: `<start.text>`
    - Telegram `chatId`: `<start.chat_id>`

    - Telegram `botToken`: `<start.botToken>`

    This creates an automatic reply system where the agent processes incoming messages and responds through Telegram.

  <Accordion title="Why is my Telegram block failing to send messages?">

    Troubleshoot common Telegram integration issues with this diagnostic checklist:

    #### Authentication Issues
    - **Invalid Bot Token**: Verify `botToken` input matches your BotFather-generated token exactly
    - **Token Format**: Ensure token follows format `123456789:ABC-DEF1234ghIkl-zyx57W2v1u123ew11`

    #### Chat ID Problems
    - **Wrong Chat Format**: User IDs are positive numbers, group/channel IDs start with `-`
    - **Bot Not Added**: For groups/channels, add your bot as an admin with message sending permissions
    - **Private Chat**: User must have started a conversation with your bot first

    #### Message Content Errors
    - **Empty Text**: Ensure `text` input is populated (not empty string or null)
    - **Text Too Long**: Telegram limits messages to 4096 characters
    - **Invalid Characters**: Some special characters may cause API errors

    #### Network and API Issues
    - **Rate Limiting**: Telegram limits bots to 30 messages per second per chat
    - **API Downtime**: Check Telegram Bot API status if all messages fail
    - **Connection Timeout**: Verify your Sim.ai workflow has stable internet connectivity

    Use `<telegram1.success>` output to conditionally handle failed sends in your workflow.
  </Accordion>
  <Accordion title="How can I create conditional Telegram notifications based on workflow results?">

    Use Telegram blocks with Condition blocks to send different messages based on workflow outcomes:

    #### Pattern 1: Success/Error Notifications
    ```
    API Call → Condition → Telegram (success)
                       → Telegram (error)
    ```

    **Configuration:**
    - Condition `input`: `<api1.success>`
    - Success Telegram `text`: `"API call completed successfully"`
    - Error Telegram `text`: `"API call failed: <api1.error>"`

    #### Pattern 2: Data-Driven Messages
    ```
    Function → Condition → Telegram (alert)
                       → (no action)
    ```

    **Setup:**
    - Function processes data and outputs `<function1.alertNeeded>`
    - Condition checks if `<function1.alertNeeded>` equals `true`
    - Telegram sends `<function1.alertMessage>` only when condition is met

    This pattern enables smart notifications that only trigger when specific conditions are satisfied.
  </Accordion>
